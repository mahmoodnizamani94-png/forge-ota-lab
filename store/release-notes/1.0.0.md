# Forge OTA Lab v1.0.0 — Stable Release

First stable release of Forge OTA Lab.

## What's Included

- **Import:** SAF file picker, share intent, and "open with" handler for .zip and .bin files
- **Format Detection:** Magic byte-based classification (ZIP, CrAU payload, legacy image)
- **Support Tiers:** Supported, Experimental, and Forensic classifications from Rust core
- **Full OTA Extraction:** Streaming REPLACE/REPLACE_XZ/REPLACE_BZ/REPLACE_ZSTD/ZERO operations
- **SHA-256 Verification:** Every extracted partition verified against manifest target_hash
- **Partition Selection:** Boot set, System analysis set, Everything extractable presets
- **Incremental Detection:** Prerequisite wizard with field-level mismatch explanations
- **Incremental Reconstruction:** Experimental tier with validated base images
- **Filesystem Browser:** Read-only ext4 browsing for verified artifacts
- **Job Persistence:** Resume after process death, history with 100-entry / 90-day retention
- **Diagnostics Export:** Anonymized bundle for failed or partial jobs
- **Settings:** Theme toggle, crash reporting consent, privileged mode gate
- **Onboarding:** Format support summary + optional telemetry consent

## Supported Formats

| Family | Tier |
|--------|------|
| Google / Pixel full payload-based OTA | Supported |
| Standard payload.bin | Supported |
| Standalone boot-critical images | Supported |
| Incremental payload-based OTA (validated base) | Experimental |
| Unknown payload-based packages | Forensic |

## Known Limitations

- Single extraction job at a time
- Samsung Odin .tar.md5 not supported (planned for v1.1)
- super.img exported as raw image only (deep unpacking in v1.2)
