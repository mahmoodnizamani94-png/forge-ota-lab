# Test Corpus — Forge OTA Lab

Synthetic test fixtures for integration testing. Generated by `corpus-gen` (Rust binary in `core-extractor-rs/corpus-gen/`).

## Regeneration

```bash
cd core-extractor-rs
cargo run -p corpus-gen
```

## Fixture Inventory

| # | File | Family | Tier | Size | Purpose |
|---|---|---|---|---|---|
| 1 | `valid_full_ota.bin` | StandalonePayload | Supported | ~100 KB | Full OTA with 2 REPLACE partitions (`boot` 64KB, `vbmeta` 4KB). SHA-256 hashes embedded for verification roundtrip. |
| 2 | `valid_incremental.bin` | StandalonePayload | Experimental | ~50 KB | Incremental payload (minor_version=5) with SOURCE_COPY op. Tests incremental detection and Experimental tier. |
| 3 | `valid_ota.zip` | AospPayloadOta | Supported | ~120 KB | ZIP containing `payload.bin` + `payload_properties.txt` + `META-INF/`. Tests sniff_seekable ZIP scanning. |
| 4 | `legacy_images.zip` | LegacyImgZip | Forensic | ~30 KB | ZIP with `boot.img` and `system.img` entries, no `payload.bin`. Tests Forensic tier classification. |
| 5 | `corrupted_manifest.bin` | — | Error | ~32 B | Valid CrAU header with garbage manifest bytes. Tests `ManifestCorrupt` / `ProtobufDecode` error path. |
| 6 | `truncated_header.bin` | — | Error | ~28 B | Valid header claiming 1024B manifest but only 4B present. Tests `ManifestCorrupt` truncation detection. |

## Expected Analysis Results

### Fixture 1 — `valid_full_ota.bin`
- **Sniff**: `MagicBytes::CrauPayload` → `StandalonePayload` → `Supported`
- **Analysis**: `is_incremental=false`, `partition_count=2`, partitions: `boot` (boot_critical), `vbmeta` (boot_critical)
- **Extraction**: Both partitions extract successfully, SHA-256 matches `target_hash` in manifest

### Fixture 2 — `valid_incremental.bin`
- **Sniff**: `MagicBytes::CrauPayload` → `StandalonePayload` → `Supported` (overridden to `Experimental` by analysis)
- **Analysis**: `is_incremental=true`, `minor_version=5`, partition `system` has `has_source_ops=true`
- **Extraction**: Blocked — SOURCE_COPY requires source partition data

### Fixture 3 — `valid_ota.zip`
- **Sniff**: `MagicBytes::Zip` → `AospPayloadOta` → `Supported` (via `sniff_seekable`)
- **Analysis**: Same as fixture 1 but accessed through ZIP entry

### Fixture 4 — `legacy_images.zip`
- **Sniff**: `MagicBytes::Zip` → `LegacyImgZip` → `Forensic`
- **Analysis**: No adapter match → Forensic mode
- **Extraction**: Not offered (no extraction CTA in Forensic mode)

### Fixture 5 — `corrupted_manifest.bin`
- **Sniff**: `MagicBytes::CrauPayload` → `StandalonePayload` → `Supported`
- **Analysis**: Fails with `PayloadError::ProtobufDecode` or `PayloadError::ManifestCorrupt`

### Fixture 6 — `truncated_header.bin`
- **Sniff**: `MagicBytes::CrauPayload` → `StandalonePayload` → `Supported`
- **Analysis**: Fails with `PayloadError::ManifestCorrupt` (file truncated)

## Constraints

- **Total size**: < 10 MB (actual < 500 KB)
- **No real OTA files**: All fixtures are synthetic (no copyright/size concerns)
- **Deterministic**: Same content on every generation (fixed patterns, no randomness)
- **Git-friendly**: Binary files committed directly (small enough)
